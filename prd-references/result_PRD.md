# Result Screen — Product Requirements Document

**Document:** `result_PRD.md`
**Version:** 1.0
**Status:** Draft
**Parent PRD:** GlamUp MVP PRD v1.0
**Scope:** Free tier view only. Premium sections (Color Analysis, Glow-Up Preview, Makeup Routine) out of scope for this document.
**Date:** May 2026

---

## 1. Purpose

The Result screen displays the AI analysis generated during the Scan. It is the primary value-delivery screen — the moment the user sees what the app produced for them. All data is loaded from Room cache. Zero network calls are made on this screen. The screen serves both free and premium users but renders only free-tier content in MVP.

---

## 2. Screen Goal

- Surface the AI analysis results in a clear, digestible layout.
- Give free users enough value to feel the product works.
- Create a clear felt gap between what is visible and what is locked — driving Paywall conversion.
- Allow the user to share their result and re-scan.

---

## 3. AI Dependency

**None at display time.** All data was generated by Gemini during the Scan and stored in Room. This screen only reads from local cache.

| Section | Data Source | Live AI call? |
|---|---|---|
| Scan date / time | Room `ScanEntity.createdAt` | No |
| Face Shape | Room `FaceAnalysis.faceShape` + `faceShapeDescription` | No |
| Skin Tone & Undertone | Room `FaceAnalysis.skinTone`, `undertone`, `undertoneDescription` | No |
| Eyes & Features row | Room `FaceAnalysis.eyeShape`, `browType`, `noseShape`, `lipType` | No |
| Celebrity Lookalikes | Room `FaceAnalysis.celebrityMatches` | No |
| Overall Score | Room `FaceAnalysis.glowUpScore` | No |
| Glow-Up Potential | Derived locally from `glowUpScore` | No |
| Share card | Canvas drawn locally | No |
| Upsell banner | Subscription state — DataStore | No |

## 4. Entry Points

| Source | NavArg passed |
|---|---|
| Scan screen — analysis complete | `faceAnalysisId: String` |
| Home — "View results" link | `faceAnalysisId: String` (latest scan id) |
| Home — Results tab (bottom nav) | `faceAnalysisId: String` (latest scan id) |
| Home — Last scan thumbnail tap | `faceAnalysisId: String` (that scan's id) |

On entry, `ResultViewModel.loadAnalysis(faceAnalysisId)` fetches `FaceAnalysis` from Room by id. If id not found (edge case — data cleared), show error state with Re-scan CTA.

On entry from Results tab with no scan history: show empty state (see Section 12).

## 6. Top Bar

**Left:** Back chevron `<` — navigates back to previous screen (Home or Scan).
**Centre:**
- Title: "Your Results"
- Subtitle: "Scan completed on {date} • {time}" — formatted from `ScanEntity.createdAt`.
  - Date format: `MMM dd, yyyy` (e.g. "May 18, 2025")
  - Time format: `hh:mm a` (e.g. "10:30 AM")

**Right:** Share icon — tapping triggers share flow (see Section 13). Same action as "Share Image" button in Section 7 of scroll content.

## 11. Section 5 — Celebrity Lookalikes Card

**Data source:** `FaceAnalysis.celebrityMatches` — array of max 3 `CelebrityMatch` objects.

**CelebrityMatch model:**
```kotlin
data class CelebrityMatch(
    val rank: Int,          // 1, 2, 3
    val name: String,       // e.g. "Zoe Saldana"
    val similarityPct: Int  // e.g. 78
)
```

**Layout:** Full-width card.

**Header row:**
- Left: "Celebrity Lookalikes" label.
- Right: "Premium" badge pill with lock icon + chevron `>`.
- Chevron tap → Paywall placeholder. `context = "celebrity"`.

**3-avatar horizontal row (equal spacing):**

Per celebrity slot:
- Rank number badge (dark circle, top-left of avatar) showing "1", "2", "3".
- Avatar: circular silhouette placeholder image (single static asset `ic_avatar_placeholder.xml`). Same placeholder for all 3. No real celebrity photos in MVP.
- Celebrity name below avatar (e.g. "Zoe Saldana").
- Match percentage below name (e.g. "78% match").

**Fallback states:**
- If `celebrityMatches` is empty or null: hide entire celebrity card. Do not show empty card.
- If fewer than 3 matches returned: show only the slots returned. No empty placeholder slots.

**No tap interaction on individual celebrity slots in MVP.** Entire card is display-only. Only the header chevron navigates.

## 12. Section 6 — Overall Score + Glow-Up Potential Row

**Layout:** Two equal-width cards side by side in a horizontal row.

### 12.1 Overall Score Card

**Data source:** `FaceAnalysis.glowUpScore`

**Contents:**
- Header: "Overall Score"
- Circular progress ring:
  - Full circle ring showing score as arc. `progress = glowUpScore / 100f`
  - Score number in centre of ring, large bold type (e.g. "86").
  - "/100" in smaller type below score number inside ring.
  - Ring drawn via `Canvas` in Compose using `drawArc`. Animates from 0 to final value on first render (600ms linear animation).
- Verdict text right of ring:
  - Verdict label (bold) + verdict body (regular). Derived locally from score. No AI call.

**Score → verdict map:**

| Score range | Label | Body |
|---|---|---|
| 90–100 | "Stunning!" | "You have exceptional natural features." |
| 75–89 | "Great!" | "You have amazing features and great glow-up potential." |
| 60–74 | "Good!" | "Solid foundation with great room to enhance." |
| 40–59 | "Building!" | "Your glow-up journey is just getting started." |
| 0–39 | "Potential!" | "Everyone starts somewhere — let's work on your glow." |

### 12.2 Glow-Up Potential Card

**Data source:** Derived locally from `FaceAnalysis.glowUpScore`. No separate AI field.

**Potential level derivation:**
```
glowUpScore 75–100 → GlowUpPotential.HIGH
glowUpScore 50–74  → GlowUpPotential.MEDIUM
glowUpScore 0–49   → GlowUpPotential.LOW
```

**Contents:**
- Header row: "Glow-Up Potential" label (left) + potential badge (right).
  - Badge label: "High" / "Medium" / "Low"
  - Badge always visible for all users (free + PRO). No gate on this section.
- Segmented bar: 5 equal segments shown as filled/unfilled pills in a row.
  - HIGH: 4 of 5 segments filled.
  - MEDIUM: 3 of 5 segments filled.
  - LOW: 1 of 5 segments filled.
  - Segments animate in left to right on first render (400ms staggered, 80ms delay per segment).
- Description text below bar:
  - HIGH: "With a few personalized tips, your glow-up potential is excellent!"
  - MEDIUM: "You have solid potential — personalized tips will make a big difference."
  - LOW: "Great foundation — consistent care will bring out your natural glow."

## 13. Section 7 — Share Result Card

**Layout:** Full-width card above sticky Re-scan bar.

**Contents:**
- Left: share icon inside circular background.
- Centre:
  - Headline: "Share your result"
  - Subtext: "Create and share your stylish result card"
- Right: "Share Image" button (dark fill, with share icon).

**Share flow:**

1. User taps "Share Image" button or top bar share icon.
2. `ResultViewModel.generateShareCard()` called.
3. Share card generated as `Bitmap` via Canvas drawing — entirely local, no network.
4. Android `Intent.ACTION_SEND` launched with `image/jpeg` MIME type.
5. User picks destination app (Instagram, WhatsApp, etc.) from system share sheet.

**Share card Canvas contents:**
- App logo / wordmark top-left.
- User's scan selfie (circular crop) top-right.
- Face shape label + shape illustration.
- Skin tone label + active swatch.
- Overall score (large number).
- Glow-Up Potential level.
- Watermark text for FREE users: "Glam Up — glamup.app" overlaid at bottom.
- PRO users: no watermark.
- Fixed card dimensions: 1080×1080px (square, Instagram-safe).

**Loading state during card generation:**
- Button shows loading indicator (circular progress, small) while Canvas drawing in progress.
- Generation should complete < 500ms. No full-screen loading overlay needed.

## 17. ResultViewModel State

```kotlin
data class ResultUiState(
    val isLoading: Boolean = true,
    val analysis: FaceAnalysis? = null,
    val subscriptionTier: SubscriptionTier = SubscriptionTier.FREE,
    val glowUpPotential: GlowUpPotential = GlowUpPotential.MEDIUM,
    val verdictLabel: String = "",
    val verdictBody: String = "",
    val isGeneratingShareCard: Boolean = false,
    val error: ResultError? = null
)

enum class GlowUpPotential { LOW, MEDIUM, HIGH }

sealed class ResultError {
    object NotFound : ResultError()
    object LoadFailed : ResultError()
}
```

**Data loading on entry:**
```
ResultViewModel.init(faceAnalysisId)
    │
    ├── FaceAnalysisDao.getById(faceAnalysisId)   → analysis
    ├── DataStore.subscriptionTier                 → subscriptionTier
    ├── derive GlowUpPotential from analysis.glowUpScore
    ├── derive verdictLabel + verdictBody from analysis.glowUpScore
    └── emit ResultUiState(isLoading=false, ...)
```

All operations in `viewModelScope`. Single `StateFlow<ResultUiState>` observed by composable.

## 18. Gemini Schema Additions Required

The following fields must be added to the Gemini JSON schema (in `scan_PRD.md` Section 16) to populate the Eyes & Features section on this screen:

| New field | Type | Example values |
|---|---|---|
| `brow_type` | string | "Defined", "Sparse", "Arched", "Straight", "Thick" |
| `nose_shape` | string | "Straight", "Wide", "Narrow", "Button", "Roman" |
| `lip_type` | string | "Full", "Thin", "Bow-shaped", "Wide", "Heart" |

These map to `BrowType`, `NoseShape`, `LipType` enums in the domain model and are stored as additional columns in `ScanEntity` in Room.

`glow_up_potential` is **not** added to the schema — derived locally from `glow_up_score` as documented in Section 12.2.

## 20. Acceptance Criteria

1. Screen loads fully from Room within 200ms on cold navigation. No loading spinner visible to user beyond initial frame.
2. Face shape SVG illustration renders correct asset for each of 7 face shape values.
3. "Learn more" bottom sheet opens correct static copy for each face shape. No network call made.
4. Active swatch in skin tone row correctly highlighted based on `skinTone` enum value.
5. All 4 feature chips (Eye, Brow, Nose, Lips) display correct values from Room. Null values show "—" without crash.
6. Celebrity card hidden entirely when `celebrityMatches` is empty or null.
7. Fewer than 3 celebrity matches renders only available slots — no empty placeholder slots.
8. Score ring animates from 0 to final score in 600ms on first render.
9. Glow-Up Potential level derived correctly from score: ≥75 = High, 50–74 = Medium, <50 = Low.
10. Segmented bar fills correct number of segments (4/5 High, 3/5 Medium, 1/5 Low) with staggered animation.
11. Verdict label and body map correctly to all five score ranges.
12. Share card generated locally as 1080×1080px bitmap. System share sheet opens successfully.
13. FREE user share card contains watermark. PRO user share card has no watermark.
14. Re-scan confirmation dialog shown before navigating away. "Cancel" dismisses dialog, stays on Results.
15. Upsell banner and feature lock banner hidden for PRO users.
16. Celebrity "Premium" badge and chevron visible for all users.
17. Empty state shown correctly when no scan history exists (Results tab entry with no data).
18. Error state shown when `faceAnalysisId` not found in Room — no crash.
19. Results tab badge (from Home nav) cleared on entry to this screen.
20. All content readable and accessible without horizontal scroll on minimum supported screen width (360dp).
