## 1. Purpose

The Feature Detail Analysis screen provides an in-depth breakdown of the user's facial features — landmark map, per-feature analysis cards with enhancement tips, symmetry score, and improvement priority list. It is a premium-only screen. All data is read from Room cache. Zero AI calls at display time.

**Scope note:** Measurements Breakdown (facial thirds percentages, golden ratio alignment), Detailed Proportions section, and Golden Ratio Comparison have been removed to reduce complexity. Screen focuses on the landmark map, 6 feature cards, symmetry score, and improvement priority.

---

## 2. AI Dependency

**None at display time.** All data generated by Gemini during Scan, stored in Room.

| Section | Data Source | Live AI call? |
|---|---|---|
| Landmark map | Room `FaceAnalysis` + ML Kit landmark positions | No |
| Feature cards (6) | Room `FeatureDetailEntity` | No |
| Symmetry score | Room `FeatureDetailEntity.symmetryScore` | No |
| Symmetry verdict | Derived locally from score | No |
| Improvement priority list | Room `FeatureDetailEntity.improvementPriority` | No |
| Expert tip | Static asset per face shape | No |

## 5. Gemini Schema — Feature Detail Fields

Additional fields required in `FULL_ANALYSIS` Gemini response:

```
feature_detail : object — {
    eye_shape_detail : {
        label        : string  — e.g. "Almond"
        status       : string  — "Balanced" | "Ideal" | "Proportional" | "Prominent" | "Defined"
        description  : string  — 1–2 sentences
        enhancement_tips: string — 1–2 sentences
    },
    brow_shape_detail : {
        label        : string  — e.g. "Soft Arch"
        status       : string
        description  : string
        enhancement_tips: string
    },
    nose_shape_detail : {
        label        : string
        status       : string
        description  : string
        contour_tips : string  — (nose uses contour_tips not enhancement_tips)
    },
    lip_shape_detail  : {
        label        : string
        status       : string
        description  : string
        enhancement_tips: string
    },
    jawline_detail    : {
        label        : string  — e.g. "Oval", "Square", "Soft"
        status       : string
        description  : string
        enhancement_tips: string
    },
    cheekbones_detail : {
        label        : string  — e.g. "High", "Medium", "Low"
        status       : string
        description  : string
        enhancement_tips: string
    },
    symmetry_score    : int     — 0–100
    symmetry_verdict  : string  — "Very Good" | "Good" | "Average" | "Excellent"
    improvement_priority: array — max 5 objects:
        {
            rank        : int
            area        : string  — e.g. "Undereye Area"
            action      : string  — 1 sentence actionable tip
        }
}
```

## 7. FeatureDetailViewModel State

```kotlin
data class FeatureDetailUiState(
    val isLoading: Boolean = true,
    val activeTab: FeatureTab = FeatureTab.OVERVIEW,
    val eyeDetail: FeatureCard? = null,
    val browDetail: FeatureCard? = null,
    val noseDetail: FeatureCard? = null,
    val lipDetail: FeatureCard? = null,
    val jawlineDetail: FeatureCard? = null,
    val cheekbonesDetail: FeatureCard? = null,
    val symmetryScore: Int = 0,
    val symmetryVerdict: String = "",
    val symmetryDescription: String = "",
    val symmetryLevel: SymmetryLevel = SymmetryLevel.AVERAGE,
    val improvementPriority: List<ImprovementPriorityItem> = emptyList(),
    val expertTip: String = "",
    val error: FeatureDetailError? = null
)

data class FeatureCard(
    val featureType: FeatureTab,
    val label: String,
    val status: String,
    val description: String,
    val tips: String,
    val illustrationAsset: String
)

data class ImprovementPriorityItem(
    val rank: Int,
    val area: String,
    val action: String
)

enum class FeatureTab {
    OVERVIEW, EYES, BROWS, NOSE, LIPS, JAWLINE, CHEEKBONES
}

enum class SymmetryLevel { EXCELLENT, VERY_GOOD, GOOD, AVERAGE }

sealed class FeatureDetailError {
    object NotFound : FeatureDetailError()
    object AccessDenied : FeatureDetailError()
}
```

**Data loading:**
```
FeatureDetailViewModel.load(faceAnalysisId)
    │
    ├── entitlement check
    ├── FeatureDetailDao.getByFaceAnalysisId(id) → entity
    ├── parse all JSON fields
    ├── derive symmetryLevel from symmetryScore (Section 12)
    ├── derive symmetryDescription from symmetryScore + verdict
    ├── load expertTip from face_shape_expert_tips.json
    │   keyed by FaceAnalysis.faceShape
    └── emit FeatureDetailUiState(isLoading=false, ...)
```

## 9. Overview Tab Content

Overview tab is the default tab on screen open. Contains:
1. Full-face landmark map card
2. Per-feature deep dive 3×2 grid
3. Symmetry score section
4. Improvement priority section
5. Expert tip

### 9.1 Full-Face Landmark Map Card

**Layout:** Full-width card. Two-column: left (description + legend), right (annotated face illustration).

**Left column:**
- "Full-Face Landmark Map" heading.
- Body text: "We detected 68 key facial landmarks to analyse your unique features and proportions."
- Legend: 5 items, each with colour dot + label:
  - Pink dot → Eyes
  - Purple dot → Brows
  - Blue dot → Nose
  - Red dot → Lips
  - Green dot → Face Contour

**Right column:**
- Static annotated face illustration: `ic_landmark_map.xml`
- Dashed annotation lines pointing to: Brow Arch, Eye Outer Corner, Nose Bridge, Nose Tip, Lip Peak, Jaw Angle, Chin Center.
- Illustration is static SVG — not generated from user's actual landmarks.
- Info icon (ⓘ) top-right of card → bottom sheet: "Landmark detection uses on-device ML to map 68 facial points across eyes, brows, nose, mouth, and jaw. This data powers your personalised analysis."

**Note:** Actual ML Kit landmark positions from scan are not re-rendered here. The illustration is a reference diagram only. Real landmark overlay exists only on the Scan screen preview.

### 9.2 Per-Feature Deep Dive Grid

3-column × 2-row grid of 6 feature cards.

**Per feature card:**
```
┌──────────────────────────┐
│ [illustration]           │
│ Eye Shape                │  ← feature type label
│ Almond                   │  ← feature label (bold)
│ [Balanced]               │  ← status badge chip
│                          │
│ Description text (2 lines│
│ max, truncated)          │
│                          │
│ Enhancement Tips         │  ← section header
│ Tips text (2 lines max)  │
│                   >      │  ← chevron
└──────────────────────────┘
```

**Feature card tap:** Navigates to that feature's dedicated tab. E.g. tapping Eye card → switches `activeTab` to `FeatureTab.EYES`.

**6 cards:**
- Eye Shape (top-left)
- Brow Shape (top-centre)
- Nose Shape (top-right)
- Lip Shape (bottom-left)
- Jawline (bottom-centre)
- Cheekbones (bottom-right)

**Illustration asset map:**
```
EYES       → ic_feature_eye.xml
BROWS      → ic_feature_brow.xml
NOSE       → ic_feature_nose.xml
LIPS       → ic_feature_lips.xml
JAWLINE    → ic_feature_jawline.xml
CHEEKBONES → ic_feature_cheekbones.xml
```

**Status badge colours (semantic only — no named colours, derive from status string):**
```
"Ideal"         → positive badge style
"Balanced"      → positive badge style
"Proportional"  → positive badge style
"Prominent"     → neutral badge style
"Defined"       → positive badge style
```
Badge style is either "positive" or "neutral" — two styles only. Mapped by status string lookup.

### 9.3 Symmetry Score Section

**Layout:** Full-width, below feature grid. Two-column.

**Left column — Score ring:**
- Circular ring, same Canvas implementation as result_PRD Overall Score.
- Score number centred (e.g. "87").
- "%" superscript beside score.
- Verdict text below ring (e.g. "Very Good").
- Animates 0 → score on first render (600ms).

**Symmetry level → verdict map:**
```
90–100 → "Excellent"   — "Your face has exceptional symmetry!"
80–89  → "Very Good"   — "Your face has excellent symmetry! Small variations make you unique."
65–79  → "Good"        — "Your face has good symmetry with natural variation."
0–64   → "Average"     — "Natural asymmetry is normal and makes you uniquely beautiful."
```

**Right column — Scale bar:**
- Horizontal gradient scale: Low → Average → High → Excellent.
- Marker dot positioned on scale based on score.
- Labels: Low / Average / High / Excellent.

### 9.4 Improvement Priority Section

**Layout:** Full-width card below symmetry score.

- Header: "Improvement Priority" + info icon.
- Info icon bottom sheet: "These areas were identified as having the highest positive impact on your overall look. Focus on these first for the best results."
- Numbered list: 5 rows max.

**Per priority row:**
```
┌─────────────────────────────────────────┐
│  [1]  Undereye Area                     │
│       Improve hydration and brightness  │
└─────────────────────────────────────────┘
```
- Rank number in filled circle (1–5).
- Area name bold.
- Action text (1 sentence).

**Rank badge styles:**
- Rank 1: largest badge style (most prominent).
- Rank 2: medium.
- Rank 3–5: standard.

### 9.5 Expert Tip

**Layout:** Full-width row at bottom of Overview tab.

- Sparkle icon left.
- "Expert Tip" label bold.
- Tip text body.

**Content source:** Static `face_shape_expert_tips.json`, keyed by `FaceAnalysis.faceShape` enum.

```json
{
  "OVAL":     "You have naturally balanced features. Small enhancements in the priority areas above will elevate your overall harmony even more!",
  "ROUND":    "Contouring along the jawline and elongating techniques will enhance your natural features beautifully.",
  "SQUARE":   "Soft, rounded makeup techniques complement your strong bone structure perfectly.",
  "HEART":    "Balance your features by drawing attention to your lips and jaw with subtle enhancement.",
  "OBLONG":   "Horizontal emphasis in brows and blush placement creates beautiful proportion.",
  "DIAMOND":  "Your prominent cheekbones are an asset — soft highlighting enhances them naturally.",
  "TRIANGLE": "Drawing attention upward with defined brows and eye makeup creates stunning balance."
```

## 13. Acceptance Criteria

1. FREE users redirected to Paywall without rendering content.
2. Overview tab shown by default on screen open.
3. Tab bar scrollable. All 7 tabs accessible. Active tab indicator updates on tap.
4. Landmark map card renders static illustration + legend with 5 correct colour dots.
5. All 6 feature cards in grid populated with correct data from Room.
6. Feature card status badge shows correct style (positive vs neutral) for each status value.
7. Truncation applied to description (2 lines) and tips (2 lines) in grid cards.
8. Tapping a feature grid card switches to that feature's tab.
9. Symmetry score ring animates 0 → score on first render.
10. Symmetry verdict and description map correctly to all four score ranges.
11. Symmetry scale bar marker positioned correctly relative to score.
12. Improvement priority list shows max 5 items. Rank 1 badge visually most prominent.
13. Expert tip text matches face shape from FaceAnalysis. All 7 face shapes covered.
14. Individual feature tabs show full (non-truncated) description and tips text.
15. Correct tip label used per feature (Contour Tips for Nose, Enhancement Tips for all others).
16. Related feature card at bottom of each tab navigates to correct next feature tab.
17. Share card generated and system share sheet opens successfully.
18. Screen loads within 200ms. All data from Room, zero network calls.

## 14. Out of Scope

- Measurements breakdown (facial thirds percentages, Eye Alignment %, Nose Symmetry % etc.) — removed for complexity.
- Detailed Proportions section (facial thirds illustration) — removed.
- Golden Ratio Comparison — removed.
- Real-time landmark re-rendering from scan data on this screen.
- Animated face scanning effect.
- Comparison with previous scan feature breakdown.
