## 1. Purpose

The Color Analysis screen displays the user's seasonal color profile derived from the Gemini scan. It is a premium-only screen. All data is read from Room cache — zero AI calls at display time. The screen tells the user which colors to wear, which to avoid, and how to apply their season to hair, makeup, and wardrobe.

---

## 2. AI Dependency

**None at display time.** All data generated by Gemini during Scan, stored in Room.

| Season + sub-season | Room `ColorAnalysisEntity.season`, `subSeason` | No |
| Season attributes | Room `ColorAnalysisEntity.attributes` | No |
| Season description | Room `ColorAnalysisEntity.seasonDescription` | No |
| Personal palette | Room `ColorAnalysisEntity.personalPalette` | No |
| Colors to avoid | Room `ColorAnalysisEntity.avoidColors` | No |
| Clothing recommendations | Room `ColorAnalysisEntity.clothingRecs` | No |
| Hair color suggestions | Room `ColorAnalysisEntity.hairColorRecs` | No |

## 5. Gemini Schema — Color Analysis Fields

These fields must be present in the Gemini `FULL_ANALYSIS` and `COLOR_ONLY` response JSON:

```
color_season        : string  — spring | summer | autumn | winter
color_sub_season    : string  — e.g. "Soft Summer", "Bright Spring"
season_attributes   : array   — 3 strings, e.g. ["Cool", "Soft", "Light"]
season_description  : string  — 1–2 sentences describing the season archetype
personal_palette    : array   — 8 hex strings
avoid_colors        : array   — 4 hex strings, with name and reason per item
avoid_reasons       : array   — 4 strings, one reason per avoid color
clothing_recs       : array   — 8 color name strings for clothing
clothing_hex        : array   — 8 hex strings matching clothing_recs
hair_color_recs     : array   — 5 objects: {name: string, hex: string}
makeup_palette      : object  — {
                          foundation: array of 3 hex strings,
                          blush: array of 4 hex strings,
                          lip: array of 6 hex strings,
                          eye_shadow: array of 4 hex strings
                        }
```

**Note on makeup palette multiple shades:** Gemini returns one primary value per category. Remaining shades in each column are derived via a **static local expansion map** — given the primary shade + undertone, a fixed set of complementary shades is appended locally. No extra API call. Map stored in `makeup_shade_expansion.json` asset.

## 7. ColorAnalysisViewModel State

```kotlin
data class ColorAnalysisUiState(
    val isLoading: Boolean = true,

    val hairColorRecs: List<HairColor> = emptyList(),

    val bestMetals: List<Metal> = emptyList(),
    val isSaved: Boolean = false,
    val isGeneratingShareCard: Boolean = false,
    val error: ColorAnalysisError? = null
)

data class PaletteColor(val hex: String, val name: String)
data class AvoidColor(val hex: String, val name: String, val reason: String)
data class ClothingColor(val hex: String, val name: String)
data class HairColor(val hex: String, val name: String)

    val foundation: List<String>,   // hex strings

    val eyeShadow: List<String>

enum class ColorSeason { SPRING, SUMMER, AUTUMN, WINTER }

sealed class ColorAnalysisError {
    object NotFound : ColorAnalysisError()
    object AccessDenied : ColorAnalysisError()

```

**Data loading:**
```
ColorAnalysisViewModel.load(faceAnalysisId)
    │
    ├── RevenueCat entitlement check → if FREE: emit AccessDenied
    ├── ColorAnalysisDao.getByFaceAnalysisId(id) → entity
    ├── parse JSON fields from entity strings
    ├── derive bestMetals from undertone (Section 13)
    ├── check SavedPaletteDao.exists(id) → isSaved
    └── emit ColorAnalysisUiState(isLoading=false, ...)
```

## 9. Section 1 — Season Result Card

**Layout:** Full-width card. Two-column internal layout — left column (illustration + season info) and right column (season selector + sub-season).

### 9.1 Left column

- Face profile illustration: static SVG asset `ic_season_profile.xml` — single reusable illustration, same for all seasons.
- "Your Best Season" label above season name.
- Season name: large bold (e.g. "Soft Summer").
- Attributes pill: single chip showing attributes joined by bullet (e.g. "Cool • Soft • Light"). Values from `attributes` array.
- Season description: 1–2 sentence body text.

### 9.2 Right column

**Season selector — 4 icons (Spring / Summer / Autumn / Winter):**
- 4 icons in a 2×2 arrangement: flower (Spring), sun (Summer), leaf (Autumn), snowflake (Winter).
- Active season icon: border highlight + label below in bold.
- Inactive seasons: icon + label, no highlight.
- **Tap behaviour — educational:** tapping any season icon opens a bottom sheet with static educational content for that season. Does NOT change the user's detected season.

**Season education bottom sheet content (static, from `season_info.json`):**
```json
{
  "SPRING": {
    "title": "Spring",
    "characteristics": "Warm, bright, and clear. Springs have golden undertones.",
    "key_colors": ["Coral", "Peach", "Warm Yellow", "Bright Green"],
    "celebrities": ["Jennifer Lopez", "Blake Lively"]
  },
  "SUMMER": { ... },
  "AUTUMN": { ... },
  "WINTER": { ... }
}
```
- Bottom sheet: season name + characteristics paragraph + key colors (4 swatch chips) + "Famous [Season] celebrities" text list.
- No navigation. Dismiss via drag down or × button.

**Sub-season chip:**
- Label: "Sub-Season"
- Chip below label: `subSeason` value (e.g. "Soft Summer").

## 10. Section 2 — Personal Color Palette Card

**Data source:** `personalPalette` — 8 `PaletteColor` objects.

**Layout:** Full-width card.

- Header: "Your Personal Color Palette"
- 8 swatches in a single horizontal row (4+4 if needed, or single 8-item row with scroll).
- Per swatch:
  - Square shape, rounded corners, 72dp × 72dp.
  - Color name below (e.g. "Dusty Rose").
  - Hex value below name (e.g. "#D8A7B1").
- Tapping a swatch: shows a tooltip/snackbar with hex value copied to clipboard. Toast: "Hex #D8A7B1 copied."

**Palette color names:**
- Returned by Gemini as `palette_color_names` array (8 strings, one per hex). Added to Gemini schema.
- Fallback if name missing: derive a generic name from hue analysis locally (e.g. hue 0–30° → "Rose", 31–60° → "Peach" etc.).

## 11. Section 3 — Colors to Avoid Card

**Data source:** `avoidColors` — 4 `AvoidColor` objects (hex + name + reason).

**Layout:** Full-width card. Two-column internal layout.

**Left column:**
- Header: "Colors to Avoid"
- 4 swatches in a row: square, 64dp × 64dp.
- Color name below each swatch.
- Hex value below name.

**Right column:**
- Info icon (ⓘ) top-right.
- Explanation text: "These colors may overwhelm your natural tones and create unflattering contrast."
- Second line: "Opt for soft, [undertone adjective], and muted shades instead." — undertone adjective derived locally: WARM → "warm", COOL → "cool", NEUTRAL → "balanced".

**Tapping info icon:** Bottom sheet with per-color reasons. Each avoid color listed with its specific reason from `avoidReasons` array.

**Layout:** Two equal-width cards side by side.

- Color name below each swatch (e.g. "Ash Brown").

### 13.2 Makeup Palette Card (right)

**Data source:** `makeupPalette` — 4 arrays of hex strings.

- Header: "Makeup Palette"

  - Foundation: 3 shades
  - Blush: 4 shades
  - Lip: 6 shades
  - Eye Shadow: 4 shades
- Shades derived via static expansion map from Gemini's primary shade + undertone. See Section 5 note.

## 16. Sticky Bottom Bar — Save Palette to Profile

**States:**

| State | Icon | Text | Behaviour |
|---|---|---|---|
| Not saved | bookmark outline | "Save Palette to Profile" / "Access your palette anytime in your profile" | Tap → save |
| Saving | loading spinner | "Saving..." | Brief, < 500ms |
| Saved | bookmark filled | "Saved ✓" | Reverts to saved state after 2s |
| Already saved | bookmark filled | "Saved to Profile" | Tap → toggle (unsave) with confirmation dialog |

**Save logic:**
```
tap Save
    │
    ├── ColorAnalysisDao.insert(colorAnalysisEntity)
    ├── update isSaved = true in UiState
    ├── bar shows "Saved ✓" for 2s
    └── bar reverts to "Saved to Profile" state
```

**Unsave confirmation dialog:**
- "Remove this palette from your profile?"
- Cancel / Remove

## 19. Acceptance Criteria

1. Screen redirects FREE users to Paywall without rendering any content.
2. Season name, sub-season, and attributes render correctly from Room entity.
3. Season selector icons show correct active state for detected season.
4. Tapping inactive season icon opens education bottom sheet with correct static content. Does not change detected season.
5. All 8 personal palette swatches render with correct hex and name.
6. Tapping a palette swatch copies hex to clipboard and shows toast.
7. All 4 avoid color swatches render with correct hex and name.
8. Info icon opens bottom sheet with per-color avoid reasons.
9. All 8 clothing rec swatches render correctly. No tap action.
10. Hair color circular swatches (5) render correctly.
11. Makeup palette columns (Foundation 3, Blush 4, Lip 6, Eye Shadow 4) render correct swatch counts from expansion map.
12. Best metals derived correctly: WARM → Gold/Rose Gold/Bronze, COOL → Silver/White Gold/Platinum, NEUTRAL → Silver/White Gold/Rose Gold.
13. Share card generated as 1080×1080px bitmap. Share sheet opens successfully.
14. Save button persists entity to Room. Bar transitions through saving → saved states.
15. Unsave confirmation dialog shown before removal.
16. Sticky bar does not overlap Share section — scroll content has correct bottom padding.
17. Empty state shown for PRO user with no color scan. CTA navigates to Scan screen with COLOR_ONLY type.
18. Screen loads fully within 200ms. All data from Room, no network calls.

## 20. Out of Scope

- Live color picker / camera-based color matching.
- Fabric texture images for clothing swatches (flat color only in MVP).
- Shopping links per color recommendation.
- Export palette as PDF or image file (share card only).
- Wardrobe integration or AR try-on.
